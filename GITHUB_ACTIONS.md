# GitHub Actions 配置指南

本指南将帮助您在 GitHub 上配置自动运行环境，实现**零成本、无人值守**的漏洞监控。

## 1. 准备工作

确保您已经将本项目代码上传到了您的 GitHub 仓库（建议设为 **Private** 私有仓库以保护您的漏洞数据）。

## 2. 配置密钥 (Secrets)

为了安全起见，我们不把密码直接写在代码里，而是使用 GitHub Secrets。

1.  打开您的 GitHub 仓库页面。
2.  点击顶部导航栏的 **Settings** (设置)。
3.  在左侧侧边栏中，向下滚动找到 **Secrets and variables**，点击展开，然后选择 **Actions**。
4.  点击右上角的绿色按钮 **New repository secret**。

请依次添加以下 3 个密钥：

| Name (名称) | Secret (值) | 说明 |
| :--- | :--- | :--- |
| `MAIL_USER` | `您的邮箱地址` | 例如 `me@gmail.com` |
| `MAIL_PASS` | `您的邮箱密码` | ⚠️ Gmail等通常需要生成 **App Password (应用专用密码)**，不是登录密码 |
| `MAIL_TO` | `接收邮箱列表` | 多个邮箱用英文逗号分隔，例如 `a@test.com,b@test.com` |

## 3. 验证运行

配置完成后，自动化流程应该不需要您做任何操作，它会：

*   **自动运行**: 每 30 分钟运行一次。
*   **手动触发**:
    1.  点击仓库顶部的 **Actions** 标签。
    2.  在左侧选择 **VulSpiderX Crawler**。
    3.  点击右侧的 **Run workflow** 按钮手动立即运行。

## 4. 查看日志

如果运行失败或没有收到邮件：

1.  在 **Actions** 页面点击最近一次的运行记录。
2.  点击 **crawl** 任务。
3.  点击 **Run Crawler** 步骤，您可以查看程序的完整控制台输出日志。

---

### 常见问题

*   **为什么第一次运行没有邮件？**
    *   如果是第一次运行且没有历史数据，程序会发送一封 "Running..." 的测试邮件。如果没有收到，请检查 `MAIL_USER` 和 `MAIL_PASS` 是否正确。
*   **如何修改运行频率？**
    *   编辑 `.github/workflows/schedule.yml` 文件。
    *   修改 `cron: '*/30 * * * *'` (每30分钟)。例如改成 `0 * * * *` (每小时)。
